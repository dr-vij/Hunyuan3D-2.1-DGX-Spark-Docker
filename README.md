# Hunyuan3D-DGX — запуск Gradio UI

Минимальная инструкция, чтобы после сборки контейнера автоматически запускался Gradio-интерфейс и был доступен из локальной сети.

## Что уже сделано в этом репо
- Dockerfile запускает `gradio_app.py` при старте контейнера и открывает порт `7860` (EXPOSE 7860).
- docker-compose.yml пробрасывает порт `7860:7860`.

Требование: внутри приложения должен существовать `/workspace/Hunyuan3D-2.1/gradio_app.py` и пакет `gradio` должен быть установлен зависимостями проекта.

## Запуск через docker compose (рекомендуется)
1. Собрать и запустить:
   - `docker compose up --build`
2. Открыть в браузере:
   - `http://<IP_вашей_машины>:7860`
   - пример: `http://192.168.1.50:7860`

## Запуск без compose
1. Сборка образа:
   - `docker build -t hy3d .`
2. Запуск контейнера (c GPU):
   - `docker run --gpus all -p 7860:7860 --name hy3d hy3d`
3. Открыть в браузере:
   - `http://<IP_вашей_машины>:7860`

## Логи и проверка
- Логи контейнера: `docker logs -f hy3d`
- Проверка, что процесс слушает порт (внутри контейнера): `ss -lntp | grep 7860`

## Типичные проблемы и решения
- Порт 7860 занят на хосте:
  - В compose поменяйте маппинг на другой внешний порт, напр.: `7870:7860`, и заходите по `http://<IP>:7870`.
- `gradio` не установлен в проекте `/workspace/Hunyuan3D-2.1`:
  - Добавьте его в зависимости проекта (requirements.txt) или установите вручную.
- Файл `gradio_app.py` отсутствует или лежит в другом месте:
  - Скорректируйте команду запуска в Dockerfile/compose на актуальный путь.

## Примечание по сети
- Узнайте IP вашей машины в локальной сети (например, `ip a` на Linux или через настройки сети).
- Убедитесь, что фаервол разрешает входящие соединения на TCP-порт 7860 (или тот, который вы пробрасываете).
